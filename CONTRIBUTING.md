# Contributing to DSI

Thank you for your interest in contributing to Dense Sourced Index! DSI is a community-driven protocol, and we welcome contributions of all kinds.

## Ways to Contribute

### 🔌 New Adapters
Create drop-in system prompt configurations for AI tools not yet supported. Place them in the `adapters/` directory following the existing format.

### 📐 Grammar Extensions
Propose new tags or syntax modifications for domain-specific use cases (e.g., medical, legal, financial). Open an issue first to discuss the proposal.

### 📊 Compression Results
Share your before/after token measurements! Create a PR adding your results to the `examples/` directory.

### 🛠️ Integrations
Build tools around DSI: VS Code extensions, GitHub Actions, CLI enhancements, etc.

## Submitting Changes

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/my-adapter`)
3. Commit your changes (`git commit -m 'Add adapter for Gemini'`)
4. Push to the branch (`git push origin feature/my-adapter`)
5. Open a Pull Request

## Protocol Change Process

Changes to the core `DSI_PROTOCOL.md` require broader discussion:

1. Open an **Issue** describing the proposed change and rationale
2. Wait for community feedback (minimum 7 days)
3. If consensus is reached, submit a PR

## Code of Conduct

- Be respectful and constructive
- Focus on the technical merits of contributions
- Welcome newcomers — everyone starts somewhere

## Questions?

Open an issue with the `question` label. We're happy to help!
